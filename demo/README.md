# demo — sample operations with reviewable I/O logs

Re-run any of these with `source env.sh && make demo` (or individually:
`make sim-demo_phy`, `make sim-demo_tinylm`, `make sim-demo_random`).
Logs land in `demo/logs/`:

- `phy_ops.log` — six narrated HyperBus operations (ID reads on both latency
  paths, zero-latency register write/readback, memory word write/readback,
  16-word burst, 64-word transfer auto-split for tCSM).
- `tinylm_generation.log` — a *trained* language model of trivial complexity
  (order-1 Markov / bigram, MLE-trained by counting on the corpus
  `"fpga works"`) embedded in the real quantised pipeline and run on the full
  system: CSR boot over uio, prompt `'f'`, 20 generated tokens decoded to
  text, bit-exact vs the numpy reference.
- `random_model_generation.log` — the seeded-random verification model,
  8 tokens autoregressive, tokens + final SSM state vs the golden trace.

The tiny LM is built by `golden/tiny_lm.py` (regenerated by `make golden`):
one-hot embeddings and a ternary bigram LM head; both Mamba layers execute
fully on real data, with W_out zeroed so the residual stream carries the
token embedding to the LM head and the learned behaviour is exact.
